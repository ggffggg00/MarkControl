О продукте

MarkControl является уникальным программным продуктом для контроля зданий и сооружений. Наш программный продукт позволяет выявить дефекты и ситуации, выходящие из под контроля до их возникновения. Благодаря этому становится возможным выявить критические деформации и причины их появления, что в свою очередь помогает своевременно принять соответствующие меры и избежать чрезвычайных ситуаций.

Данный проект имеет открытый исходный код, лицензированный Apache License v.2

 

Минимальные системные требования

  • Windows 8 или старше / .NET Framework 4 или старше
  • Клавиатура и мышь
  • 1GB RAM
  • Любой монитор, шириной более 1100 пикселей

 

Начало работы

Основные принципы

Программный продукт поддерживает работу с несколькими проектами. Каждый проект хранится отдельно друг от друга во внутренней памяти программы. Создание нового проекта происходит в окне приветствия, где и выводятся все созданные ранее проекты. Все данные хранятся в единой базе данных, скрытой от использования другими ПО.

Первый запуск

При первом запуске программы нас встречает окно приветствия, предоставляющее возможность создать новый проект

 

Создание проекта

Ввод основной информации о проекте

При создании проекта необходимо указать все необходимые данные для работы над ним:

  • Название проекта
  • Количество отметок для измерения высот
  • Количество структурных блоков в объекте
  • Выбрать файл изображения
  • Распределить марки по структурным блокам

 

Важно! Для схемы проекта поддерживается исключительно PNG файлы (возможно наличие альфа канала)

 

Распределение марок по структурным блокам

Распределение марок происходит вручную, для этого необходимо нажать кнопку Распределить марки

Важно! В составе всех блоков должно находиться одинаковое количество марок

При распределении марок необходимо поочередно кликать на их номера, при заполнении определенного блока. После заполнения блока марками, программа автоматически переключится на следующий блок.

В случае ошибки при вводе необходимо нажать на кнопку Сброс значений, после чего ввести заново марки в блоки.

После распределения будет доступна кнопка Готово. По ее нажатию окно закрывается и основное окно принимает введенную схему распределения.

Важно! Именование блоков в программе происходит на латинском алфавите заглавными буквами

Важно! На данном этапе распределения марок по блокам кнопка К следующему блоку не реализует функциональных возможностей, поскольку в каждом блоке должно быть одинаковое количество марок

Пример: Для записи марок в блок А мы смотрим на схему объекта и видим марки 1, 2, 12. Значит поочередно кликаем на марки этого блока в списке слева

 

Создание проекта

После успешного ввода всех необходимых данных мы увидим примерно такую картину

Вся основная информация введена и можно создавать проект нажатием на соответствующую кнопку. В случае отмены действия вы вернетесь на окно приветствия.

После создания проекта он автоматически открывается в Главном окне. Вся дальнейшая работа будет происходить именно в нем.

 

Открытие существующего проекта

Если вы уже создавали проект, то в окне приветствия сбоку вы увидите список созданных вами проектов. Необходимо кликнуть на существующий проект для продолжения работы с ним

 

 

Основной функционал

Обзор интерфейса главного окна

При открытии только что созданного или пустого проекта мы можем видеть такую картину

Опишем разделы интерфейса

  1. Список разделов программы (в пустом проекте отображается только Обозреватель данных)
  2. Содержание выбранного раздела
  3. Строка состояния

Строка состояния включает в себя:

  • Индикатор актуальности данных в базе данных
  • Количество эпох на данный момент
  • Текущая точность измерений Е
  • Текущий коэффициент сглаживания А
  • Единица измерения углов Alpha
  • Название текущего проекта
  • Кнопка закрытия проекта и выхода на экран приветствия

 

Обозреватель данных

Описание раздела

Обозреватель данных - это раздел программного продукта, в котором реализована возможность манипулирования данными. Только в этом разделе возможно изменить какие-либо данные. При изменении любых данных происходит полный перерасчёт в остальных разделах и буферизация данных в программе

Если индикатор в панели имеет зеленый цвет, значит все данные сохранены. В противном случае необходимо сохранить изменения в БД чтобы не потерять их при перезапуске программы или проекта.

В случае отсутствия значений отметок высот марок будет выведено соответствующее сообщение и скрыты разделы уровней декомпозиции

 

 

 

Действия с данными

Изменение значений высотных марок

Для изменения значений высотных марок необходимо дважды кликнуть на необходимую ячейку таблицы и ввести обновленное значение. Для применения изменений необходимо нажать клавишу Enter. Для отмены введенных изменений необходимо нажать клавишу Escape.

 

Импорт значений высотных марок

Импорт необходим для добавления значений высотных марок, в том числе в большом количестве или для добавления значений в только что созданный проект.

Программа принимает для импорта CSV файлы с запятой-разделителем.

Важно! Количество столбцов в CSV файле должно быть равно количеству марок в проекте

Для импорта значений высотных марок необходимо нажать кнопку Импорт значений, выбрать файл для импорта и открыть его. Программа автоматически его прочитает и запишет значения марок в память программы. Для сохранения значений в БД необходимо нажать кнопку Применить изменения.

 

Экспорт значений высотных марок

Если необходимо выгрузить все значения высотных марок в csv файл, можно воспользоваться функцией экспорта. Для этого необходимо нажать на кнопку Экспорт значений и выбрать расположение файла для сохранения.

 

Добавление прогнозной эпохи

В программе имеется возможность добавить эпоху на основе текущих значений марок. Для этого необходимо нажать кнопку Добавить эпоху. Данные будут добавлены в конец таблицы значений высот, но не будут записаны в БД. Для сохранения изменений в базе данных необходимо нажать Применить изменения.

 

Изменение коэффициента А и точности Е

Вы можете изменять коэффициент экспоненциального сглаживания (А) и величину СКО (Е) в метрах. После ввода нужного значения необходимо нажать Применить напротив форму ввода. Все данные в разделах будут пересчитаны, но изменения не будут записаны в БД. Для записи измененных данных нажмите кнопку Применить изменения

 

Сортировка по столбцу

В случае необходимости у пользователя есть возможность отсортировать записи по значениям определенного столбца. Для этого необходимо кликнуть на заголовок столбца.

 

Первый уровень декомпозиции

Описание раздела

Раздел первого уровня декомпозиции позволяет отследить динамику изменения положения объекта рассматривая его как единое целое. Для перехода на данный раздел необходимо иметь загруженные отметки высот марок и кликнуть на соответствующий раздел в меню слева.

 

Таблица расчётов

М, М+, М-

Значение М характеризует величину поступательного движения исследуемого объекта

Значение М+ характеризует максимальный порог допустимого поступательного движения объекта с учетом указанной величины СКО

Значение М- характеризует минимальный порог допустимого поступательного движения объекта с учетом указанной величины СКО

 

Alpha, Alpha+, Alpha-

В программе данное расчётное значение выводится в виде угловых секунд

Значение Alphaхарактеризует величину вращательного движения исследуемого объекта

Значение Alpha+ характеризует максимальный порог допустимого вращательного движения объекта с учетом указанной величины СКО

Значение Alpha- характеризует минимальный порог допустимого вращательного движения объекта с учетом указанной величины СКО

 

M pr

Характеризует прогнозное значение М с учетом коэффициента экспоненциального сглаживания. Значения предсказываются на эпоху вперед, причем прогнозная эпоха выделяется желтым цветом

 

Alpha pr

Характеризует прогнозное значение Alpha с учетом коэффициента экспоненциального сглаживания. Значения предсказываются на эпоху вперед, причем прогнозная эпоха выделяется желтым цветом

 

Состояние допуска

В случае нахождения фазовых значений в рамках допустимого программа выводит True, в противном случае выделяет строку красным цветом и пишет в столбце False

 

Состояние устойчивости

В случае наличия в таблице хотя бы одной неустойчивой эпохи выводится сообщение о предпочтительности проведения следующего уровня декомпозиции

 

График траекторий

Ниже таблицы представлен график траекторий, на котором располагаются четыре серии, каждую из которых можно выключить с помощью флажка справа. При наведении курсора на одну из вершин графика можно получить необходимые значения в данной точке. На графике представлены траектории:

  • Фазовая траектория ( (А) - (М) )
  • Прогнозная траектория ( (А pr) - (M pr) )
  • Нижняя граница траектории ( (A-) - (М-) )
  • Верхняя граница траектории ( (A+) - (М+) )

 

Второй уровень декомпозиции

Описание раздела

Раздел второго уровня декомпозиции позволяет отследить динамику изменения положения объекта рассматривая его как отдельно взятые блоки. Для перехода на данный раздел необходимо иметь загруженные отметки высот марок и кликнуть на соответствующий раздел в меню слева.

 

Исследование блока

По сути выполняются все те же операции, что и в первом уровне декомпозиции, только над отдельно взятым блоком сооружения. Для начала исследования необходимо выбрать исследуемый блок в выдвижном списке справа. После будут загружены расчётная таблица и график с выводом.

 

Боковая панель

В боковой панели располагается график Фазовых траекторий ( (А) - (М) ) всех блоков. Для включения или отключения серий графиков необходимо кликнуть соответствующий флажок в меню сверху графика. Ниже графика представлена схема объекта для упрощения ориентирования по структурным блокам.

 

Третий уровень декомпозиции

Описание раздела

Третий уровень декомпозиции позволяет разбить уже имеющиеся структурные блоки на подблоки благодаря анализу связей марок внутри него. Для начала работы необходимо выбрать блок сооружения для разбора.

 

Таблица разностей превышений

В таблице содержатся разности превышений марок для каждой связи в каждой эпохе. Связь в эпохе считается твердой, если разница превышений не превышает заданное значение СКО. В таком случае ячейка окрашивается в зеленый цвет, в противном - в красный.

 

Граф связей

Граф связей является визуализацией связей между марками блоков. Зеленые ребра графа показывают твердые связи, красные, в свою очередь, говорят об отсутствии жесткости связи. Граф подчинен физике, поэтому его вершины можно перемещать и располагать в нужном порядке.

 

Разбиение на подблоки

После изучения таблицы разности превышений пользователь может разбить структурный блок на подблоки. Для этого необходимо в форме справа указать количество подблоков и нажать Разобрать. После откроется уже знакомое окно распределения марок по блокам. После распределения в разделе появится список подблоков в меню справа, будет отрисован график фазовых траекторий подблоков и сверху появится вкладка Анализ.

Для включения выключения серий на графике фазовых значений используются флажки в правом меню.

Важно! При обновлении данных в обозревателе данных распределение марок по подблокам сбрасывается

 

Анализ подблока

Для анализа подблока необходимо выделить подблоки. После появления вкладки Анализ сверху необходимо в нее перейти. Изначально вы увидите пустую страницу, но стоит кликнуть на подблок (не на флажок) в списке справа, как появится уже знакомая форма анализа блока.

Важно! При обновлении данных в обозревателе данных распределение марок по подблокам сбрасывается, а вкладка Анализ скрывается

 

Четвертый уровень декомпозиции

Описание раздела

Раздел предоставляет возможность проследить динамику изменения высот марок в зависимости от эпохи.

 

Включение и отключение серий

В правом боковом меню имеется возможность по клику на необходимые флажки отображать нужные серии марок.

 

Как узнать значения в вершине

При наведении курсора мыши на точку графика будет отображена высота и эпоха марки

 

Работа с проектами

Переключение между проектами

Для переключения между проектами необходимо покинуть уже открытый проект нажатием на кнопку Закрыть проект в нижнем правом углу После показа окна приветствия вы можете выбрать другой проект или создать новый. Как это сделать описано выше в разделе Начало работы

 

Решение проблем

Я изменил данные, но не записал изменения в БД. Как вернуть актуальные данные?

Необходимо всего лишь закрыть проект и открыть его снова

Как создать дамп базы данных?

В папке размещения программы необходимо найти файл marksdb и скопировать его в надежное место. В случае утраты файла его можно вернуть на место и все данные будут восстановлены.

Программа не хочет импортировать значения марок, что делать?

Скорее всего вы выбрали CSV файл неверного формата. Программа поддерживает файлы с разделителем в виде запятой и разделителем дробной части в виде точки.
